================================================================================
                           What Was Created
================================================================================

A fully functional React frontend for a context-aware hearing aid companion app
with 6 main screens, real-time transcription, interactive ML feedback, and
smart alert notifications.

KEY COMPONENTS:
- 12 React components (6 screens + 4 UI + 2 layout)
- 1 custom WebSocket hook with auto-reconnect
- 30+ API integration functions
- 1414 lines of carefully crafted CSS with dark theme
- Complete state management with React Context

================================================================================
                        The 6 Screens Explained
================================================================================

1. HOME SCREEN (🏠)
   Purpose: Overview and quick access
   Features:
   - Device status card (battery, connection)
   - Context banner (location, time, calendar)
   - Quick action buttons (4 cards)
   - How it works pipeline visualization
   - Recent alerts with reasoning
   - Test Alert button for simulation

2. ALERTS SCREEN (🔔)
   Purpose: Sound monitoring and context rules
   Features:
   - Toggle monitored sound types (6 types)
   - Fire alarm always locked on (safety-critical)
   - Context rules cards (Meeting, Sleep, Outdoors, Restaurant)
   - Alert history with confidence scores
   - Real-time API updates

3. TRANSCRIBE SCREEN (📝)
   Purpose: Real-time speech-to-text conversion
   Features:
   - Animated waveform visualization
   - Start/Stop recording controls
   - Live transcript with speaker labels
   - Language/Speakers/Font size info
   - Download and copy transcript options
   - Periodic line fetching during recording

4. TRAIN AI SCREEN (🤖) [The Key Feature]
   Purpose: Interactive Machine Learning feedback training
   Features:
   - Stats bar (Confirmed, Corrected, Accuracy %)
   - Pending sound classifications with confidence badges
   - Three-button feedback: "Yes correct", "No it was...", "Skip"
   - Correction panel with sound options (grid selection)
   - Already Reviewed section (dimmed history)
   - SPECTRA model training reference
   - Direct API submission of feedback

5. SETTINGS SCREEN (⚙️)
   Purpose: Audio environment and hearing program control
   Features:
   - Noise gauge (semicircle, dB reading, color-coded)
   - Hearing program selector (5 programs: Home, Restaurant, Music, Outdoors, Sleep)
   - Fine tuning sliders (3 controls with debounced API calls)
   - Real-time environment classification
   - Program settings persistence

6. PROFILE SCREEN (👤)
   Purpose: User information and device settings
   Features:
   - Profile header with avatar gradient
   - Device settings group (3 items)
   - Smart alerts group (3 items)
   - ML & Personalization group (model stats)
   - Footer with version info and institution credit

================================================================================
                       Technical Architecture
================================================================================

COMPONENT HIERARCHY:
App
├─ AppContextProvider
│  └─ AppContent
│     └─ PhoneFrame
│        ├─ StatusBar
│        ├─ [CurrentScreen] (based on activeTab 0-5)
│        ├─ TabBar
│        └─ NotificationOverlay

STATE MANAGEMENT:
Global Context (AppContext):
- activeTab (0-5) - Current screen
- notification - Alert popup data
- deviceStatus - Device info
- setActiveTab, showNotification, hideNotification functions

Component Local State:
- Data fetching (loading, error, data arrays)
- UI state (expanded, selected items)
- Form state (slider values, toggle states)

DATA FLOW:
User Action → Event Handler → API Call → State Update → Re-render

WEBSOCKET FLOW:
WebSocket Server → Message → useWebSocket Hook → lastMessage → App.jsx
→ showNotification() → NotificationOverlay renders

API INTEGRATION:
- Base URL: /api (proxied to http://localhost:3001)
- 30+ functions in services/api.js
- All functions: error handling, try-catch, JSON parsing
- Components handle errors with error states

================================================================================
                         How to Get Started
================================================================================

STEP 1: Install Dependencies
$ cd client
$ npm install

STEP 2: Start Development Server
$ npm run dev
Output: http://localhost:5173

STEP 3: Ensure Backend is Running
Backend must be on: http://localhost:3001
Check these endpoints:
- GET /api/device/status
- GET /api/profile
- WS /ws (for alerts)

STEP 4: Open in Browser
Navigate to: http://localhost:5173
You'll see the HearClear app in a phone-sized frame

STEP 5: Test Features
- Click tabs to switch screens
- Click Test Alert on Home screen
- Toggle sounds on Alerts screen
- Adjust sliders on Settings screen
- Submit feedback on Train AI screen

STEP 6: Build for Production
$ npm run build
Output: dist/ directory (ready to deploy)

================================================================================
                       API Endpoints Used
================================================================================

ALERTS & MONITORING:
GET /api/alerts                          - Fetch all alerts
GET /api/alerts/{id}                     - Get single alert
POST /api/alerts/simulate                - Create test alert
GET /api/monitored-sounds                - Get sound toggles
PATCH /api/monitored-sounds/{type}       - Toggle sound type

CONTEXT & RULES:
GET /api/context-rules                   - Fetch context rules
PATCH /api/context-rules/{id}            - Toggle rule

INTERACTIVE ML:
GET /api/iml/pending                     - Get unreviewed classifications
GET /api/iml/reviewed                    - Get past feedback
GET /api/iml/stats                       - Get accuracy statistics
POST /api/iml/feedback                   - Submit feedback

HEARING PROGRAMS:
GET /api/programs                        - List programs
POST /api/programs/{id}/select           - Select program
PATCH /api/programs/{id}/settings        - Update settings
GET /api/environment                     - Current environment

PROFILE & DEVICE:
GET /api/profile                         - User profile
PATCH /api/profile                       - Update profile
GET /api/device/status                   - Device info

TRANSCRIPTION:
POST /api/transcription/start            - Start recording
POST /api/transcription/{id}/end         - End recording
GET /api/transcription/{id}/lines        - Get transcript lines
GET /api/transcription/sessions          - List sessions

WEBSOCKET:
WS /ws                                   - Real-time alerts

================================================================================
                        Customization Guide
================================================================================

CHANGE COLORS:
Edit src/styles/app.css CSS variables:
:root {
  --primary: #6C5CE7;     ← Your color
  --accent: #00CEC9;      ← Your color
  ...
}

ADD A NEW SCREEN:
1. Create src/components/screens/NewScreen.jsx
2. Import in src/App.jsx
3. Add to screens array
4. Add tab in TabBar
5. Add CSS styles in app.css

MODIFY API ENDPOINT:
Edit src/services/api.js:
const API_BASE = '/api'  ← Change here

CHANGE PORT:
Edit vite.config.js:
server: {
  port: 5173  ← Change here
}

CUSTOMIZE THEME:
All styling in src/styles/app.css:
- CSS variables at top
- Components below
- Animations at end
- Media queries at bottom
